# Plan: Project A.C.T. — Contact Us Page

## Design Identity (studied from existing site)
- **Accent:** `--red: #D90429` (eyebrows, underline hovers, progress bar, footer corner).
- **Themes:** dark navy (`#001b3a` / `#001024`, white text) on standalone pages like `archive.html`; light (`#f0f2f5`/`#fff`) on the homepage.
- **Fonts:** Playfair Display (display headings, italic `<em>` accents) + Inter (body, tiny uppercase letter-spaced eyebrows/labels).
- **Recurring patterns:** giant low-opacity "ghost" background text; `reveal` scroll-in animations + number counters; fixed blurred top nav with a red scroll-progress bar; shared footer with logotype, mission blurb, contact line, and social icons.
- **Closest template:** `archive.html` — a standalone, dark-themed content page. The Contact page should mirror its nav + footer exactly for consistency.

## Assumptions (defaults chosen — confirm via plan_exit follow-up)
1. **Theme:** Dark navy, matching `archive.html`.
2. **Form behavior:** Static mockup — client-side validation + inline success confirmation. No backend. Easy to wire to a service later.
3. **Contact details:** Reuse footer info (Revital Healthcare (EPZ) Ltd., Kilifi County, Kenya, website) + clearly-marked **placeholder** email/phone for the owner to edit.
4. **Nav:** Add a "Contact" link to the `archive.html` nav (and `index.html`) pointing to `contact.html`; mark it `active` on the contact page.

## Files
- **Create** `contact.html` — the page (mirror `archive.html` head/fonts/nav/footer structure).
- **Create** `css/contact.css` — page-specific styles on top of the archive token set.
- **Edit** `archive.html` — add `<li><a href="contact.html">Contact</a></li>` to `.nav-links`.
- **Edit** `index.html` — add Contact to `.nav-links` (same pattern) for cross-linking.

## Page structure (`contact.html`)
1. **Head:** same font/preconnect/favicon as `archive.html`; link `css/contact.css`.
2. **Nav:** copy `archive.html` nav verbatim; add Contact link; set `active` on it. Keep `#siteNav` + `#navProgressIndicator`.
3. **Hero** (mirror `archive.html` hero): `hero-ghost` "CONTACT", eyebrow "Project A.C.T. — Get In Touch", Playfair title `Let's Build<br><em>Impact.</em>`, short desc.
4. **Contact grid** (two columns on desktop, stacked on mobile):
   - **Left — details card:** rows for Address, Email (placeholder), Phone (placeholder), Website, and social icons (reuse CSR LinkedIn `project-a-c-t-revital-healthcare-csr` + Instagram `project_a.c.t_` from `index.html` footer). Each row: small red uppercase label + value.
   - **Right — form:** fields Name (required), Email (required, validated), Organization (optional), Subject (required), Message (required, textarea). Styled inputs with red focus border, Playfair/Inter labels, red submit button (`<em>`-free, uppercase letter-spaced). On submit: `preventDefault`, validate, show success panel ("Message received — we'll be in touch."), reset form.
5. **Footer:** copy `archive.html` `<footer class="section-footer">` block verbatim (ghost ACT, mission, contact, social, bottom bar).
6. **Scripts:** inline nav-scroll + progress indicator + `reveal` IntersectionObserver (same as `archive.html`). No counters needed unless KPIs are added.

## Styling notes (`css/contact.css`)
- Reuse archive tokens (`--navy`, `--navy-dark`, `--red`, `--white`, `--line-muted`).
- Inputs: `background: rgba(255,255,255,0.04); border:1px solid var(--line-muted); border-radius:10px; color:#fff;` focus → `border-color: var(--red); box-shadow:0 0 0 3px rgba(217,4,41,0.15)`.
- Labels: Inter 8px uppercase `letter-spacing:0.2em`, color `rgba(255,255,255,0.5)`.
- Submit button: solid `--red`, white uppercase text, hover lift + glow (mirror `.footer-social-link:hover` / `.nav-cta` feel).
- Reuse `.reveal`/`.delay-1`/`.delay-2` classes; add `.contact-grid`, `.contact-detail`, `.contact-form`, `.field`, `.form-success` classes.
- Responsive: single column ≤768px; nav already responsive.

## Validation
- Open `contact.html` in a browser; verify nav progress bar, hero ghost text, reveal animations.
- Submit empty form → validation errors shown; submit valid form → success panel + reset.
- Resize to mobile width → grid stacks, nav intact.
- Confirm footer/social links match other pages; Contact link active and present in `archive.html`/`index.html`.
